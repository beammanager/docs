## Database setup

<Callout type="info">
  All commands in this section require root permissions, so run them using
  `sudo` or `doas` if you are not the root user.
</Callout>

### MariaDB installation

```bash tab="Ubuntu/Debian"
apt update
apt install mariadb-server mariadb-client
```

```bash tab="Fedora/RHEL/Alma"
dnf install mariadb-server mariadb
```

```bash tab="openSUSE"
zypper install mariadb mariadb-client
```

```bash tab="Alpine"
apk add mariadb mariadb-client
```

### Start and enable MariaDB

```bash tab="Systemd"
systemctl start mariadb
systemctl enable mariadb
```

```bash tab="OpenRC"
rc-service mariadb start
rc-update add mariadb default
```

### Database Installation

<Callout type="warn">
  If you already have MariaDB setup, you should probably skip this step.
</Callout>

Run this command if you want to install the database somewhere other than the default location

```bash
mariadb-install-db --datadir=<path>
```

Then, secure the database installation

```bash
mariadb-secure-installation
```

It will first prompt you for the password of the root database user, just press enter as that is not set.  
Then, it will ask you if you want to set a password for the root account. Choose no. This is still secure because the account is linked to the root user, if they are not acting as the root user on your system, they cannot access the MariaDB root account.

### Database creation

Enter the mariadb shell

```bash
mariadb
```

Create the database

```sql
CREATE DATABASE BeamManager;
```

### Database user creation

<Callout type="warn">**Remember the password!**</Callout>

```sql
-- [!code word:password here]
CREATE USER beammanagerapi IDENTIFIED BY 'password here';
```

Now, exit the mariadb shell

```sql
EXIT;
```

## Docker installation

Follow the [official guide](https://docs.docker.com/engine/install/)  
For AlmaLinux, follow the RHEL instructions  
For openSUSE, follow [these instructions](https://en.opensuse.org/Docker)  
For Alpine, follow [these instructions](https://wiki.alpinelinux.org/wiki/Docker)

Then, start and enable docker

```bash tab="Systemd"
sudo systemctl start docker
sudo systemctl enable docker
```

```bash tab="OpenRC"
doas rc-service docker start
doas rc-update add docker default
```

Then, add yourself to the docker group.

```bash
sudo usermod -a -G docker $USER
```

You will need to loug out and log back in for the change to take effect.

## Backend configuration

Create the configuration file

```bash
mkdir $XDG_CONFIG_HOME/beammanager
touch api.toml
```

Then add this to the configuration file:

<Callout type="info">
  Now you have to use the password you *should* have remembered from earlier
</Callout>

```toml
# [!code word:password from earlier]
dbPassword = "password from earlier"
```
